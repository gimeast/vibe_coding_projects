PDF 생성 라이브러리 선택

리포트 PDF를 어떻게 만들까요?

A — 권장

Puppeteer (HTML → PDF)

작동 방식
HTML 템플릿 → Chromium 렌더링 → PDF 저장
웹 UI와 동일한 CSS로 PDF 스타일링 가능

장점

  • ✅ 한글 폰트 자동 지원 (Windows 시스템 폰트 사용)
  • ✅ CSS로 자유로운 레이아웃 — BMW M 디자인 그대로 적용
  • ✅ 표, 배지, 색상 등 복잡한 UI 완벽 렌더링
  • ✅ 웹 UI와 PDF 스타일 공유 가능

단점

  • Chromium 바이너리 포함으로 용량 큼 (~300MB)
  • 첫 실행 시 다운로드 필요
B

@react-pdf/renderer (코드 기반)

작동 방식
React PDF 컴포넌트 → PDF 직접 생성
PDF.js 기반, Chromium 불필요

장점

  • 가볍고 빠름 (Chromium 불필요)
  • 브라우저에서 미리보기 가능

단점

  • ⚠️ 한글 폰트 수동 임베딩 필요 (TTF 파일 직접 로드)
  • ⚠️ CSS 서브셋만 지원 — flexbox 제한적
  • 별도 PDF 전용 컴포넌트 따로 작성 필요
  • 복잡한 레이아웃 구현 난이도 높음
💡 추천: 한글 뉴스 + 복잡한 테이블 레이아웃이 많은 리포트이므로 A (Puppeteer)가 압도적으로 유리합니다.
Windows에서 Malgun Gothic 등 시스템 한글 폰트를 자동으로 씁니다.
